# Security Policy

## Reporting a Vulnerability

The zkproof team takes security seriously. We appreciate your efforts to responsibly disclose your findings.

### How to Report

If you discover a security vulnerability, please report it by:

1. **DO NOT** open a public issue
2. Send an email to the maintainers with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: We will acknowledge receipt of your report within 48 hours
- **Updates**: We will keep you informed about the progress of the fix
- **Credit**: We will credit you for the discovery (unless you prefer to remain anonymous)

### Supported Versions

Currently, security updates are provided for the latest version on the main branch.

## Security Best Practices

When using zkproof:

- Keep dependencies up to date
- Review all code changes before deployment
- Use proper authentication and authorization
- Follow the principle of least privilege
- Enable audit logging where applicable
- Regularly review security advisories

## Disclosure Policy

- Security issues are typically disclosed after a fix is available
- We aim to coordinate disclosure with reporters
- CVEs may be requested for significant vulnerabilities
